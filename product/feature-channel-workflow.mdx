## Why this exists

Decisions get made in Slack and never make it back into the docs. The spec drifts from reality, onboarding suffers, and "what did we decide?" becomes archaeology. This workflow keeps the **doc as the source of truth** by giving every feature a dedicated channel and a living spec, and having Claude reconcile the two: when the conversation changes a decision, Claude opens a pull request that updates the spec.

## The shape of it

<Steps>
  <Step title="Spin up a feature channel">
    Create a Slack channel for the feature and tag Claude in it. That tag is the signal that the channel is in scope. The channel is where the feature gets discussed, decided, and refined.
  </Step>
  <Step title="Pair it with a spec">
    Each channel maps to exactly one spec page in the [`almanaclabs/docs`](https://github.com/almanaclabs/docs) repo at `product/<feature>.mdx`. Either hand Claude the starting doc, or ask Claude to draft one from the conversation so far. Bookmark the spec (and its open PR) in the channel so the source of truth is one click away.
  </Step>
  <Step title="Claude follows along">
    The team discusses the feature normally. Claude stays present and reads the conversation, staying quiet unless there's something to capture.
  </Step>
  <Step title="Claude reconciles the doc, proactively">
    When a decision changes the spec, Claude opens a **draft PR** against `almanaclabs/docs` on its own that edits the relevant page to match. No prompting needed. The PR description links back to the Slack thread that drove the change, so every doc edit is traceable to its conversation.
  </Step>
  <Step title="A human reviews and merges">
    Claude picks a reviewer based on who drove the decision in the thread and requests their review. Claude never self-merges. The reviewer requests changes in-thread or on the PR, then merges. On merge, Mintlify auto-deploys the updated spec.
  </Step>
</Steps>

## When Claude acts

Claude runs **proactively**: it watches each feature channel for decisions that change or extend the current spec, and opens a PR without being asked. You can still nudge it explicitly when you want a specific change captured:

- **@mention Claude** in the channel with what changed, or
- React to the deciding message with :memo:.

Either way the output is the same: a **draft PR, one per discrete change**, that a human reviews and merges.

## PR conventions

- **One PR per discrete change**, kept in draft until a human is ready to review.
- Every PR description links the Slack thread that prompted it.
- Claude requests review from whoever drove the relevant decision, and keeps an eye on its own PRs, surfacing CI failures or review comments back in the channel.
- Once merged, the doc is the source of truth. If Slack and the doc disagree, the doc wins, and a PR should fix whichever is wrong.

## What's needed to turn this on

<CardGroup cols={2}>
  <Card title="Claude listening in feature channels" icon="ear-listen">
    Claude has to be present and responsive in each feature channel for the proactive flow to work. Tag it in when the channel is created.
  </Card>
  <Card title="Docs repo access" icon="github">
    Claude needs write access to `almanaclabs/docs` to open PRs. Already set up.
  </Card>
  <Card title="A channel-to-spec mapping" icon="signs-post">
    One channel, one spec page at `product/<feature>.mdx`. Bookmark the spec in the channel so the mapping is unambiguous.
  </Card>
  <Card title="A reviewer per change" icon="user-check">
    Claude selects a reviewer from the conversation, but someone has to actually review and merge so PRs don't pile up.
  </Card>

## Decisions

The settled defaults for this workflow:

| Question | Decision |
|----------|----------|
| Where specs live | `product/<feature>.mdx` in `almanaclabs/docs` |
| How a channel enters scope | The team creates the channel and tags Claude |
| Starting doc | Provided by the team, or drafted by Claude from the conversation |
| When Claude opens PRs | Proactively, when a decision changes the spec |
| PR shape | One PR per discrete change, draft until reviewed |
| Reviewer | Claude selects based on who drove the decision |
